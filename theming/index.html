Getting Started

Populr theming is based on a couple of existing open-source technologies. If you already know Handlebars.js and LESS, feel free to skip ahead. Otherwise, keep reading intrepid explorer!

A Populr theme, like any (good) html/css breakdown, is separated into two layers — first semantic structure / content, and second, style.

Structure / Content

A theme's structure is built in Handlebars.js, a javascript-based templating language. Although it may appear overwhelming at first, a handlebars template is simply centered around one basic construct: the curly-brace {% raw %}{ }{% endraw %}. With this, you can define every piece of stucture and content you will use in creating your theme.

Example Asset Template (don't worry about the sytax right now)

{% raw %}
{{#asset "text" size="2"}}
<h3 class="asset-title">{{{title}}}</h3>
<div class="content">
  {{{body}}}
</div>
{{/asset}}{% endraw %}

The Theme Skeleton

Once you are comfortable with Handlebars.js and LESS, download the Populr Theme Skeleton to get started with your own theme. Populr themes are simple HTML5 websites with a few special hooks. Let's look at an simple theme to get a feel for the syntax:

Sample Theme index.html

{% raw %}
<!doctype html>
<html>
<head>
  <meta charset="utf-8">
  <title>My New Theme</title>
  <link rel="stylesheet/less" type="text/css" href="/styles/main.less">
  <script src="http://populr.me/assets/developers/themer.js"></script> 
  <script src="/js/main.js"></script>
</head>
<body>
  <script id="theme" type="text/x-handlebars-template">
    {{{title}}}
    {{#region "isotope" id="main-region" gutter_width="20" dummy_content="I3,T,T,E,D4"}}
      {{#asset "text" size="m"}}
        <h3>{{title}}</h3>
        <div class="content">
          {{{body}}}
        </div>
      {{/asset}}

      {{#asset "documentgroup" size="m"}}
        <h3>{{title}}</h3>
        <div class="content">
          <ul>
          {{#each contents}}
            <li><a href="{{original_url}}">{{source_file_name}}</a></li>
          {{/each}}
          </ul>
        </div>
      {{/asset}}

      {{#asset "embedgroup" size="m"}}
        {{#each contents}}
          <div class="embed">{{{html}}}</div>
        {{/each}}
      {{/asset}}

      {{#asset "imagegroup" size="m"}}
        {{{imageslider contents width="310" height="232"}}}
      {{/asset}}

      {{#asset "videogroup" size="m"}}
        {{{videoplayer contents width="310" height="232" resolution="medium"}}}
      {{/asset}}
    {{/region}}
  </script>
</body>
</html>
{% endraw %}

Anatomy of a Theme

A Populr theme is divided up into two main sections:

Regions

At it's core, a region is simply a container for assets. The magic happens when defining what type of layout engine a region uses to display those assets. We'll talk more about this in a bit. The two layout engines currently available are:

  • Columnizer
  • Isotope

Assets

Assets are the heart of Populr. Each item of content in a Pop is one of five asset types:

  • Text
  • Image(s)
  • Recorded Video(s)
  • Document(s)
  • Embed(s)

Each asset is contained inside of a region, and is rendered according to the region it resides in, as well as the size selected for the asset.

Regions

Regions are intelligent layout containers for assets. Defining a region is done by using the {% raw %}{{#region}}{% endraw %} handlebars.js block helper. Every region you define will become an area users can place assets in the Pop editor. Regions are rendered as html5 section elements.

  • Template Code
  • Compiled Output
{% raw %}
{{#region "isotope" id="main-region" gutter_width="20"}}
{{/region}}{% endraw %}
{% raw %}
<section class="region isotope-region" id="main-region" 
         data-region-type="isotope" data-region-id="0"></section>{% endraw %}

There are a few settings that every region should have.

Name Usage
id The id of the region
max_auto_assets

The maximum number of assets Populr will automatically place in this region when a user switches to your theme. When automatically placing assets, Populr will work through your regions in source order. If it finds a region that doesn't define max_auto_assets, or if the value is 0, it will add all of the remaining assets to that region.

Heads up! You should always have at least one region available as a catch-all for any remaining assets. This setting is optional.

Isotope Layout Engine

The Isotope engine packs assets vertically based on the column a user places them in. In this layout, a user can move an asset's location between any of the logical columns. The engine then moves all of the assets in that column up. A gutter separates all of the assets and columns.

Helper

{% raw %}
{{#region "isotope" id="main-region" column_width="120" gutter_width="10"}}
{{/region}}{% endraw %}

Engine-Specific Settings

Name Usage
column_width

The width of the smallest asset size

Example: column_width="300"
gutter_width

The amount of space between assets

Example: gutter_width="20"
sizes

The asset sizes available. Must be a comma-delimited list and can contain one or more of the following sizes:

  • s (small)
  • m (medium)
  • l (large)

Example: sizes="s,m,l"

Asset Sizing

Asset sizes are determined by the user in isotope regions. There are three sizes that a user can select: s, m, or l. You must define the available sizes in the sizes setting on the region block helper.

Columnizer Layout Engine

The Columnizer engine arranges assets in columns and rows. As a user adds assets to a row, the engine will push on new columns and resize existing columns to fit the width of the row. As the themer, you define the maximum number of columns per row.

Helper

{% raw %}
{{#region "columnizer" id="main-region" max_cols="3"}}
{{/region}}{% endraw %}

Engine-Specific Settings

Name Usage
max_cols

The maximum number of columns (assets) in a row

Example: max_cols="3"

Asset Sizing

Asset sizes are determined by the number of columns in the columnizer region. As the number of columns change, so does each asset size in the row. For example, if you have one asset in a row, then that asset's size is 1 and it's class will be asset-size-1. If you have two assets in a row, each asset receives a size of 2 and their classes will be asset-size-2.

Assets

Assets are the actual content a user places in Pop. Each type of asset has unique properties that you can access when creating your theme. Defining an asset is done by using the {% raw %}{{#asset}}{% endraw %} handlebars.js block helper. Assets are rendered as div elements.

  • Template Code
  • Compiled Output
{% raw %}
{{#asset "text" size="m"}}
  <div class="title-wrapper">
    {{{title}}}
  </div>
  <div class="content-wrapper">
    {{{contents}}}
  </div>
{{/asset}}{% endraw %}
{% raw %}
<div class="asset asset-type-text asset-size-m" data-region-id="0">
  <div class="asset-inner">
    <div class="title-wrapper">
      <h2 class="title">Aenean Commodo Ligula</h2>
    </div>
    <div class="content-wrapper">
      <div class="content">
        <p>Lorem ipsum dolor sit amet, consectetuer adipiscing elit.</p>
      </div>
    </div>
  </div>
</div>{% endraw %}

Heads Up Asset templates must be nested inside of region definitions, and you must define asset templates for every asset type and every available asset size in each region.

There are a few settings that every asset template should have.

Name Usage
size The size this asset template is associated with.

Text Asset Type

The text asset type is the most basic asset in Populr. It allows the user to enter a title and body content.

Template Helpers

Populr provides several addition Handlebars.js helpers which will assist you in creating your theme.

Resize

The {% raw %}{{{resize}}}{% endraw %} helper is fundamental to Populr theming. Users can upload any size image they want in the Populr editor. Obviously, this is not ideal for a good looking theme, so the resize helper allows you to scale and crop images to your liking by rewriting urls so that they pass though our image transformer. 

{% raw %}
{{{resize original_url width=640 height=480}}}
{% endraw %}

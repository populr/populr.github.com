Getting Started

Populr theming is based on a couple of existing open-source technologies. If you already know Handlebars.js and LESS, feel free to skip ahead. Otherwise, keep reading intrepid explorer!

A Populr theme, like any (good) HTML/CSS breakdown, is separated into two layers — first semantic structure / content, and second, style.

Structure / Content

A theme's structure is built in Handlebars.js, a JavaScript-based templating language. Although it may appear overwhelming at first, a Handlebars.js template is simply centered around one basic construct: the curly-brace {% raw %}{ }{% endraw %}. With this, you can define every piece of stucture and content you will use in creating your theme. The Handlebars.js elements of the asset return HTML, so they can be wrapped in div tags to apply styling.

Example Asset Template (don't worry about the syntax right now)

{% raw %}
{{#asset "text" size="2"}}
<div class="asset-title">{{{title}}}</div>
<div class="content">
  {{{body}}}
</div>
{{/asset}}{% endraw %}

Asset templates are comprised of 3 parts: a title, the content, and a description. (Text assets like the one shown above generally don't include a description, since the content is a block of text.) The content is the only mandatory component of the template; titles and descriptions are optional. These parts can be appear in any order within the asset template.

Style

Stylesheets are built in LESS CSS. If you are comfortable with CSS, you will be comfortable with LESS. Populr uses LESS variables to allow users to change various settings in the Pop. Refer to the variables.less file in the theme skeleton for more information.

Heads up! If you choose not familiarize yourself with Handlebars.js and LESS before continuing, you will have a difficult journey ahead. Seriously, read up on these two technologies or we will be forced to flip a table   (╯°□°）╯︵┻━┻   You don't want us to do that, do you?

The Theme Skeleton

Once you are comfortable with Handlebars.js and LESS, download the Populr Theme Skeleton to get started with your own theme. Populr themes are simple HTML5 websites with a few special hooks. Let's look at a simple theme to get a feel for the syntax.

Sample Theme index.html

{% raw %}
<!doctype html>
<html>
<head>
  <meta charset="utf-8">
  <title>My New Theme</title>
  <link rel="stylesheet/less" type="text/css" href="/styles/main.less">
  <script src="http://populr.me/assets/developers/themer.js"></script> 
  <script src="/js/main.js"></script>
</head>
<body>
  <script id="theme" type="text/x-handlebars-template">
    {{{title}}}
    {{#region "packer" id="main-region" columns="1" gutter_width="20" max_auto_assets="0" sizes="m" default_size="m" dummy_content="I-3-m,T-1-m,T-2-m,E-1-m,D-4-m"}}
      {{#asset "text" size="m"}}
        {{{title}}}
        <div class="content">
          {{{body}}}
        </div>
      {{/asset}}

      {{#asset "documentgroup" size="m"}}
        {{{title}}}
        <div class="content">
          <ul>
          {{#documents}}
            <li><a href="{{original_url}}">{{source_file_name}}</a></li>
          {{/documents}}
          </ul>
        </div>
        {{{description}}}
      {{/asset}}

      {{#asset "embedgroup" size="m" width="310"}}
        {{{title}}}
        <div class="content">
          {{#embeds}}
            <div class="embed">{{{html}}}</div>
          {{/embeds}}
        </div>
        {{{description}}}
      {{/asset}}

      {{#asset "imagegroup" size="m"}}
        {{{title}}}
          <div class="content">
            {{{imageslider contents width="310" height="232"}}}
          </div>
        {{{description}}}
      {{/asset}}

      {{#asset "videogroup" size="m"}}
        {{{title}}}
          <div class="content">
            {{{videoplayer contents width="310" height="232" resolution="medium"}}}
          </div>
        {{{description}}}
      {{/asset}}
    {{/region}}
  </script>
</body>
</html>
{% endraw %}

Anatomy of a Theme

A Populr theme is basically made up of two things: regions and assets.

Regions

At its core, a region is simply a container for assets. The magic happens when defining what type of layout engine a region uses to display those assets. We'll talk more about this in a bit. The two layout engines currently available are:

  • Packer
  • Columnizer

Assets

Assets are the heart of Populr. Each item of content a user adds to a pop is one of five asset types:

  • Text
  • Image(s)
  • Recorded Video(s)
  • Document(s)
  • Embed(s)

Each asset is contained inside of a region, and is rendered according to the region it resides in, as well as the size selected for the asset.

Regions

Regions are intelligent layout containers for assets. Every region you define will become an area users can place assets in the Pop editor.

Regions are defined with the {% raw %}{{#region}}{% endraw %} Handlebars.js block helper. They are rendered as HTML5 section elements.

  • Template Code
  • Compiled Output
{% raw %}
{{#region "packer" id="main-region" max_auto_assets="0" columns="2" column_width="300" gutter_width="20" sizes="m,l"}}
{{/region}}{% endraw %}
{% raw %}
<section class="region packer-region" id="main-region" 
         data-region-type="packer" data-region-id="0" data-gutter_width="20">
</section>{% endraw %}

There are a few settings that should be defined for every region.

Name Usage
id The ID of the region
max_auto_assets

The maximum number of assets Populr will automatically place in this region when a user switches to your theme. When automatically placing assets, Populr will work through your regions in source order. If it finds a region that doesn't define max_auto_assets, or if the value is 0, it will add all of the remaining assets to that region. This setting is optional, but recommended.

Heads up! Every theme should have at least one region set up with max_auto_assets="0" to act as a catch-all for any remaining assets.

Packer Layout Engine

The Packer engine packs assets vertically based on the column a user places them in. In this layout, a user can move an asset's location between any of the logical columns. The engine then moves all of the assets in that column up. A gutter separates all of the assets and columns.

Helper

{% raw %}
{{#region "packer" id="main-region" columns="3" column_width="120" gutter_width="10" sizes="s,m,l"}}
{{/region}}{% endraw %}

Engine-Specific Settings

Name Usage
columns

The number of columns in the region

Example: columns="2"
column_width

The width of the smallest asset size in pixels

Example: column_width="300"
gutter_width

The amount of space between assets in pixels

Example: gutter_width="20"
sizes

The asset sizes available. Must be a comma-delimited list and can contain one or more of the following sizes:

  • s (small)
  • m (medium)
  • l (large)

Example: sizes="s,m,l"

Asset Sizing

Within Packer regions, users can choose the size for each asset. So if a user sets an asset to, for example, the smallest size possible, that asset's size is s and it's assigned the class asset-size-s.

The valid sizes are specified in the sizes setting on the {% raw %}{{#region}}{% endraw %} block helper. You can make up to three sizes available: s, m, and l.

Columnizer Layout Engine

The Columnizer engine arranges assets first into rows, then into columns. Each row spans the full width of the region. When a user adds an asset to a row, the engine automatically resizes other assets in that row to make room for the new asset. As the themer, you define the maximum number of columns per row.

Helper

{% raw %}
  {{#region "columnizer" id="main-region" max_cols="3"}}
  {{/region}}{% endraw %}

Engine-Specific Settings

Name Usage
max_cols

The maximum number of columns (assets) in a row

Example: max_cols="3"

Asset Sizing

Within Columnizer regions, asset sizes are determined by the number of assets in the row. For example, if a row contains just one asset, then that asset's size is 1 and it's assigned the class asset-size-1. If a row contains two assets, each asset is a size 2 and they're each assigned the class asset-size-2.

Assets

Assets are the actual content a user places in Pop. Each type of asset has unique properties that you can access when creating your theme. Assets are defined with the {% raw %}{{#asset}}{% endraw %} Handlebars.js block helper, and are rendered as div elements.

Heads up! Asset templates must be nested inside of region definitions, and you must define asset templates for every asset type and every available asset size in each region.

  • Template Code
  • Compiled Output
{% raw %}
{{#asset "text" size="m"}}
  <div class="title-wrapper">
    {{{title}}}
  </div>
  <div class="content-wrapper">
    {{{contents}}}
  </div>
{{/asset}}{% endraw %}
{% raw %}
<div class="asset asset-type-text asset-size-m" data-region-id="0">
  <div class="asset-inner">
    <div class="title-wrapper">
      <h2 class="title">Aenean Commodo Ligula</h2>
    </div>
    <div class="content-wrapper">
      <div class="content">
        <p>Lorem ipsum dolor sit amet, consectetuer adipiscing elit.</p>
      </div>
    </div>
  </div>
</div>{% endraw %}

The size setting must be defined for every asset template.

Name Usage
size

The size this asset template is associated with.

For a Packer region, the available asset sizes are s, m, and l, and the asset sizes valid for the region are specified in the {{#region}} helper.
Packer example: size="m"

For a Columnizer region, the available asset sizes are 1, 2, 3, and so on.
Columnizer example: size="2"

Asset Template Variations

Each region must contain asset templates for every combination of asset type and size that can appear in the region. This provides an opportunity to apply different layouts and styling to assets depending on parent region, asset type, or size. Furthermore, the {% raw %}{{{if_many}}}{% endraw %} helper can be used to apply layout and style variations depending on how many items are contained in an asset - see the {% raw %}{{{if_many}}}{% endraw %} helper documentation for details.

Get creative! The most successful Populr themes are the ones that give users a range of stylistic variations. Add a region at the top of the theme for a hero image or video. Vary your typography depending on the size of the asset. Change your image slider parameters for the body and footer regions. Mix things up a bit and you'll have a beautiful theme on your hands in no time.

Template Helpers

Populr provides several additional Handlebars.js helpers which will assist you in creating your theme.

Resize

The {% raw %}{{{resize}}}{% endraw %} helper is fundamental to Populr theming. Users can upload any size image they want in the Populr editor. Obviously, this is not ideal for a good looking theme, so the resize helper allows you to scale and crop images to your liking by redirecting urls through our image transformer.

Helper

{% raw %}
  {{{resize original_url width=640 height=480}}}
  {% endraw %}

Settings

Name Usage
width Target width optional
height Target height optional

Heads up! Defining either a width or a height will proportionally scale the image to fit the defined dimension. If you define both the width and the height, the image will be scaled and cropped to fit within both dimensions. A value of 0 will act the same as an undefined value.

If Many

The {% raw %}{{{if_many}}}{% endraw %} helper gives you the ability to detect whether a group asset has one item or multiple items in its contents. This is particularly useful when you want to create one template for a single item, and another for multiple. This helper works exactly the same as a normal Handlebars.js if helper. The else block is optional.

Helper

{% raw %}
  {{#if_many images}}
    <!-- loop through images -->
  {{else}}
    <!-- display single image -->
  {{/if_many}}
  {% endraw %}

Settings

There are no settings for this helper.

Format File Size

The {% raw %}{{{format_file_size}}}{% endraw %} helper formats a filesize, listed in bytes, to a more-readable representation. It will automatically decide if the size should be displayed as one of the following: 'Bytes', 'KB', 'MB', 'GB', 'TB'

Helper

{% raw %}
  {{{format_file_size source_file_size precision=2}}}
  {% endraw %}

Settings

Name Usage
precision Number of decimal places (optional, default is 2)

Image Slider

The {% raw %}{{{imageslider}}}{% endraw %} helper creates a ready-made JavaScript image slider for your imagegroup assets.

If there is only one image in an asset's {% raw %}{{{images}}}{% endraw %} array, then that image will not be cropped; it will simply be scaled to fit the full width of its container. However, if there are multiple images in the {% raw %}{{{images}}}{% endraw %} array, they will be first scaled, then cropped as needed to fit the dimensions specified by the width and height parameters.

Helper

{% raw %}
  {{{imageslider images width=640 height=480}}}
  {% endraw %}

Settings

Name Usage
width Target width
height Target height
effect Transition effect. Can one of the following: sliceDown, sliceDownLeft, sliceUp, sliceUpLeft, sliceUpDown, sliceUpDownLeft, fold, fade, random, slideInRight, slideInLeft, boxRandom, boxRain, boxRainReverse, boxRainGrow, boxRainGrowReverse. (optional, default is fade)
delay Time (in milliseconds) each slide should remain in place (optional, default is 5000)

Heads up! Unlike a simple {% raw %}{{{resize}}}{% endraw %} helper, you should always define both the width and the height of an image slider.

Video Player

The {% raw %}{{{videoplayer}}}{% endraw %} helper creates a ready-made HTML5/Flash fallback video player for videogroup assets. The player uses the open-source Projekktor project. If there are multiple videos in a group, this helper will create a thumbnail playlist below the player.

Helper

{% raw %}
  {{{videoplayer videos width=640 height=480 resolution="medium}}}
  {% endraw %}

Settings

Name Usage
width Target width
height Target height
resolution The resolution category the player should use. Can one of the following: small, medium, large. The value should generally reflect the relative size of the asset you are templating. (optional, default is medium)

Styling

The CSS styling for Populr themes is generally stored in 3 LESS files: main.less, variables.less, and preboot.less.

Main.less

Main.less is where the majority of your CSS should be stored. It contains calls to import variables.less and preboot.less at the beginning of the file.

Variables.less

Variables.less stores LESS variables that can be changed by the user through Populr's pop editing interface, or that are changed by the pop rendering engine when the page loads.

Name Description
@base-url

The pop's base URL

This variable is used to set relative links within a theme definition. For example, the background image for a page element can be set via CSS like this:

{% raw %}background: transparent url("@{base-url}/images/region-bg.png") no-repeat;{% endraw %}
@accent-color

The accent color used in the pop. This variable is frequently used in Populr's most basic themes to set the color of text links, but it can be used to set the color of any theme element.

This value can be overridden by the user with the Color drop-down menu.

@background-color

The background color used when this page is first rendered

Once a user sets a background image for their pop, Populr will calculate the average color of the image and store that value in @background-color. When someone opens the pop, Populr will display this color as the pop background while the background image is being loaded. This technique creates a less abrupt transition when the background image is finally loaded and displayed.

@headline-font-family

The default font family for the pop title and asset titles when the theme is selected

This value can be overridden by the user with the Font drop-down menu.

@headline-font-weight

The default font weight for the pop title and asset titles when the theme is selected

This value can be overridden by the user with the text formatting bar.

@text-color The color of body text

Preboot.less

Preboot.less is, to quote its creator Mark Otto, "a super awesome pack of mixins and variables to be used in conjunction with LESS, a CSS preprocessor for faster and easier web development." Read about Preboot.less on Mark's website.

Preboot is not mandatory to include with your theme. We used Preboot because it helped us put together some of our themes faster. You're welcome to use it or not as you see fit.

System-Generated Classes

Populr's theming engine automatically applies system-generated classes to certain page elements. Here are a few that will be especially valuable for building unique behaviors into your theme, such as applying styling variations based on content location, size, etc.

Rows in Columnizer Regions

Each columnizer row will automatically have both of the following classes applied.

  • columnizer-row
  • columnizer-cols#, where # is the number of assets contained in the row

 

Assets in Columnizer Regions

Each asset in a columnizer row is assigned to a class according to its index in the row.

  • columnizer-col#, where # is the index number of the asset in the row. The first asset has an index of 1, the second an index of 2, and so on.

 

Asset Types

Every asset is assigned to a class according to its type. Only one of the following classes is applied per asset.

  • asset-type-text
  • asset-type-imagegroup
  • asset-type-videogroup
  • asset-type-documentgroup
  • asset-type-embedgroup

 

Asset Size

Every asset is assigned to a class according to its size. Only one of the following classes is applied per asset.

Assets in Packer Regions

  • asset-size-s
  • asset-size-m
  • asset-size-l

Assets in Columnizer Regions

  • asset-size-1
  • asset-size-2
  • asset-size-3
  • ...and so on

Building Locally

The Populr theming engine was developed to enable talented designers and developers like you to work on your theme locally before uploading it to Populr. To illustrate theme behavior outside of the pop editor interface, a theme can be loaded with dummy content from Populr's servers. The theme can then be run locally on your machine using a simple HTTP server, such as the one built into Python.

Loading Your Theme with Dummy Content

Dummy content is generic asset content that's displayed in your theme when you run your theme locally in your browser. The text is lorum ipsum text, the images are from lorumpixel, the embed is one of the world's greatest short films, etc.

Dummy content is defined as an attribute of the {% raw %}{{region}}{% endraw %} Handlebars.js helper. You might have noticed this earlier while reviewing the sample theme. Here it is again: 

{% raw %}
{{#region "packer" id="main-region" columns="1" gutter_width="20" max_auto_assets="0" sizes="m" default_size="m" dummy_content="I-3-m,T-1-m,T-2-m,E-1-m,D-4-m"}}{% endraw %}

Each comma-separated string of the dummy content represents a single asset. The syntax for each string is as follows:
[asset type]-[# of items]-[asset size]

[asset type] can have the following values:

  • T for text assets
  • I for image assets
  • V for video recording assets
  • D for document assets
  • E for embed assets

[# of items] represents something different based on the asset type.

  • For text assets, it represents the number of paragraphs of lorum ipsum text that will be generated.
  • For all other assets, it represents the number of items that are contained within the asset. For example, a piece of dummy content defined as I-3 represents an image asset containing 3 images.

Dummy content is defined slightly differently for Packer regions and Columnizer regions.

Packer Dummy Content

Dummy content for Packer regions must specify a size for each asset. The available values are s, m, and l.

Columnizer Dummy Content

Dummy content for columnizer regions does not specify a size. Instead, the | (pipe) character is used to specify the end of a row, and asset sizes are determined based on the number of assets in a row. For example, dummy content defined as "T-1,T-1|I-3 represents two text assets in the first row, each with a size of 2 and containing a single paragraph of lorum ipsum text, and single image asset in the next row with a size of 1 and containing 3 images.

Running Your Theme Using a Simple Python HTTP Server

To see how your theme looks when it's populated with your dummy content, run an HTTP server locally on your machine and use it to host your theme. The Populr team uses Python to create a server, since it comes with a simple HTTP server built right in.

To run a server on your Mac using Python, open Terminal, then use the cd command to change to your theme's directory. Once you're there, use this command to start your server: 

{% raw %}
python -m SimpleHTTPServer 8080

You can then check out your theme by opening a browser and entering the URL http://localhost:8080.
